# REST API

The backend exposes a versioned REST API under the `/api/v1` prefix on port **3001**. Interactive, always-current documentation (with request/response schemas) is generated by Swagger at:

**http://localhost:3001/api/docs**

The tables below summarize the available endpoints by module.

## Orders — `/api/v1/orders`

| Method | Path | Purpose |
| --- | --- | --- |
| `GET` | `/orders` | List orders |
| `POST` | `/orders` | Create an order |
| `GET` | `/orders/:id` | Get one order |
| `PATCH` | `/orders/:id/status` | Update order status |

**Create order body** (`CreateOrderDto`): `merchantId`, `customerId`, `amount` (decimal string, e.g. `"100.000000"`), and optional `merchantAddress`, `customerAddress`, `tokenAddress`.

## Payments — `/api/v1/payments`

| Method | Path | Purpose |
| --- | --- | --- |
| `POST` | `/payments/request` | Generate a payment request (derives the on-chain `orderId`) |
| `GET` | `/payments/order/:orderId` | Get the payment for an order |
| `GET` | `/payments/:id` | Get a payment by id |
| `POST` | `/payments/:id/confirm` | Confirm a payment |

**Payment request body** (`CreatePaymentRequestDto`): `platformOrderId` (keccak256-hashed into the on-chain `orderId`), `merchantAddress`, `customerAddress`, `amount` (decimal string).

## Disputes — `/api/v1/disputes`

| Method | Path | Purpose |
| --- | --- | --- |
| `POST` | `/disputes` | Open a dispute |
| `POST` | `/disputes/:id/evidence` | Submit evidence (IPFS hash) |
| `GET` | `/disputes/:id` | Get a dispute |
| `POST` | `/disputes/:id/appeal` | Appeal a ruling |

## Oracle — `/api/v1/oracle`

| Method | Path | Purpose |
| --- | --- | --- |
| `POST` | `/oracle/delivery` | Submit a delivery signal |
| `GET` | `/oracle/status/:orderId` | Oracle/consensus status for an order |
| `POST` | `/oracle/mock/deliver/:orderId` | Test shortcut to simulate delivery |

## Settlement — `/api/v1/settlement`

| Method | Path | Purpose |
| --- | --- | --- |
| `GET` | `/settlement/batches` | List committed settlement batches |
| `POST` | `/settlement/trigger` | Manually trigger a batch settlement |
| `POST` | `/settlement/verify` | Verify a Merkle inclusion proof |

## Fiat — `/api/v1/fiat`

| Method | Path | Purpose |
| --- | --- | --- |
| `GET` | `/fiat/rate` | Current exchange rate (e.g. USDT ↔ VND) |
| `POST` | `/fiat/on-ramp` | Fiat → token (mock) |
| `POST` | `/fiat/off-ramp` | Token → fiat (mock) |

## Metrics — `/api/v1/metrics`

| Method | Path | Purpose |
| --- | --- | --- |
| `GET` | `/metrics` | System metrics |

## Oracle service (separate, port 3002)

The standalone oracle service exposes carrier webhooks and mapping endpoints — see [Simulating a Delivery](../usage/simulating-delivery.md) for the full list.
